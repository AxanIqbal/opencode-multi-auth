## OVERVIEW

OpenCode plugin that overrides provider `openai` with OAuth-backed multi-account rotation. It translates Chat Completions requests to ChatGPT Codex Responses API calls, buffers SSE, and returns fabricated `chat.completion` JSON.

## CONVENTIONS

- ESM only: package has `type: module`; TypeScript uses `moduleResolution: NodeNext` and explicit `.js` import suffixes.
- Node built-ins use `node:` imports; types use `import type` where possible.
- Source of truth is `src/`; `dist/` is build output and package payload.
- Provider ID intentionally equals `openai`, replacing the built-in provider.
- OAuth only for real accounts. `DUMMY_API_KEY` exists because OpenCode provider plumbing expects an API key.
- Config is env-only through `envConfig()` plus `DEFAULT_CONFIG`; vars use `OPENCODE_MULTI_AUTH_*`.
- Rate limits are per-model by default. `OPENCODE_MULTI_AUTH_PER_MODEL=0` switches to global cooldowns.
- Account file saves are immediate after cooldown, quota, add/remove, and token refresh changes; `save()` first merges disk state from other processes.
- Toasts are best-effort and debounced per account for 5 seconds.
- Request fingerprint headers mimic Codex CLI: `user-agent`, `x-client-request-id`, optional `x-codex-installation-id`, `traceparent`.

## ANTI-PATTERNS (THIS PROJECT)

- Do not add generic fetch passthroughs around this plugin. All provider traffic must route through `customFetch` so rotation works.
- Do not add response formats beyond Chat Completions input to Codex Responses output to fabricated Chat Completions JSON.
- Do not treat `resets_at` as always-relative seconds. Parser must distinguish relative seconds, epoch seconds, epoch ms, and date strings.
- Do not remove the retry fallback header in `retryAfterResponse`; OpenCode needs a retry signal even when no account reset is known.
- Do not skip `save()` after mutating account health, quota, or token state. Multiple processes share the same account file.
- Do not weaken `consecutiveFailures >= 3`; disabled accounts must stay out of selection and earliest-reset calculations.
- Do not hand-edit `dist/`; rebuild with TypeScript.
- Do not assume `src/index.ts` is a thin entrypoint; it is intentionally a fat controller until refactored.
